# MemFS 需求说明

这次我要实现的是一个简单的内存文件系统 MemFS。它不需要真的把数据写到磁盘上，而是把文件和目录都保存在内存里。这个实现主要用于说明我理解 rCore 风格的文件系统分层，并且代码需要能够通过 `make` 编译。

MemFS 需要支持最基本的文件操作：创建文件、创建目录、通过路径打开文件、读文件、写文件、移动文件 offset、截断文件、列出目录项、删除目录项，以及关闭文件描述符。最简单的验收流程是：打开 `/hello`，写入 `abc`，把 offset 移回 0，再读出来，结果应该还是 `abc`。

设计上，系统调用层不能直接操作文件内容，而应该先通过 fd 找到当前进程 fd table 里的文件对象。每次 `open` 都会产生一个新的 file handle，这个 file handle 保存自己的 offset 和读写权限；真正的文件内容则保存在 inode 里。因此，同一个文件被打开两次时，两个 fd 应该共享同一个 inode，但它们的 offset 应该彼此独立。

内部结构上，我需要实现一个 `Inode` 抽象接口，再用 `MemInode` 表示内存中的普通文件或目录。普通文件用 `Vec<u8>` 保存内容，目录用映射表保存“文件名到 inode”的关系。因为 inode 内容可能被多个 file handle 共享，所以共享状态需要用锁保护。

这个 MemFS 不追求完整 POSIX 语义，也不需要实现真实磁盘、block cache、journal、硬链接、软链接或复杂权限系统。它的重点是把“fd table、file handle、inode、文件内容”这几层关系写清楚，并且能够通过简单测试证明 read/write/open/seek 这些核心路径是可用的。
